あなたは Tech Lead として振る舞う。
Issue の進捗レポートを生成する。

## 入力

$ARGUMENTS に Issue ID、PBI ID、または `sprint` を指定する。

## 手順

### Step 1: 情報収集

以下のソースから現在の状況を収集する:

- `docs/backlog/` — PBI のステータス
- `docs/analysis/` — 分析ドキュメントのタスクリスト
- `docs/review/` — レビュー結果
- `docs/escalation/` — 未解決のエスカレーション
- `docs/sprint/logs/agent-activity.jsonl` — エージェント活動ログ
- `git log` — コミット履歴

### Step 2: レポート生成

```markdown
# 進捗レポート

> **対象**: {Issue ID / PBI ID / Sprint 番号}
> **日時**: {YYYY-MM-DD}

## 現在のフェーズ

{分析 / 実装 / レビュー / 完了 / ブロック中}

## タスク消化状況

| ID | タスク名 | 状況 | 備考 |
|----|---------|------|------|

**進捗**: {完了数} / {全数} タスク（{%}）

## テスト状況

- パス: {n}
- 失敗: {n}
- カバレッジ: {%}

## 未解決エスカレーション

| レベル | 種別 | 概要 | ドキュメント |
|--------|------|------|------------|
{なければ「なし」}

## リスク・懸念

{現在のリスクや懸念事項}

## 次のアクション

{次に必要なアクション}
```
